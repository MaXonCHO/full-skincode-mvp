# SkinCode Admin Tool

Отдельная статическая панель для мониторинга базы тональных средств и ручного управления связками.

## Как запускать

```bash
cd admin-tool
python -m http.server 4173
```

Либо раздайте через любой другой статики-сервер (Vercel, Netlify, S3). Панель не взаимодействует с пользовательским сайтом и требует только сеть до backend API.

## Конфигурация

- **API URL** — по умолчанию `http://127.0.0.1:8000`. Можно ввести любой будущий домен (например, `https://admin-api.skincode.ru`).
- **Admin token** — совпадает с переменной `ADMIN_API_TOKEN` на backend. На dev окружении используется `dev-admin-token`. Хранится в `localStorage` браузера и не связывается с основным приложением.

## API Требования

Панель ожидает следующие эндпоинты (реализованы в `backend/main.py`):

- `GET /admin/stats` — агрегаты по базе.
- `GET /admin/gaps` — список продуктов без связок.
- `GET /admin/links` — таблица связей, поддерживает фильтры `brand` и `line`.
- `POST /admin/links` — создание/обновление связи.
- `PATCH /admin/links/{id}` — изменение веса.
- `DELETE /admin/links/{id}` — удаление.

Все запросы требуют заголовок `X-Admin-Token`.

## Перенос базы на отдельный сервер

Чтобы переподключить панель к новой базе, достаточно:
1. Задеплоить backend (или отдельный админ-API) с теми же эндпоинтами.
2. Задать `ADMIN_API_TOKEN` в переменных окружения API.
3. В панели обновить URL и токен в правом верхнем блоке — данные сохранятся локально.

Никаких прямых ссылок на основной сайт или фронтенд SkinCode нет.
